## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
As a Compliance Officer, I want to generate evidence packages so audit stakeholders can review traceable operational history.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 Packages can be generated by date range, organization, site, product, or trace ID
- [ ] #2 Packages include audit events, feed metadata, quarantine records, and related inventory movements
- [ ] #3 Package wording states that the export supports review and does not certify compliance
- [ ] #4 Evidence package artifact is a ZIP file containing manifest.json plus JSON exports for audit events, feeds, quarantine records, movements, and included object references
- [ ] #5 manifest.json includes package ID, generated_at, generated_by, filter criteria, trace IDs, source object references, content hashes, and review disclaimer
- [ ] #6 Evidence package object key follows evidence-packages/{organizationId}/{yyyy-mm-dd}/{packageId}.zip
- [ ] #7 Package generation stores metadata in the database and emits an audit event with filters and artifact URI
- [ ] #8 Package retrieval is restricted to compliance/platform-admin roles and retrieval attempts are audited
<!-- AC:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
1. Define evidence package request, database metadata, object key convention, ZIP layout, and manifest schema before implementation.
2. Implement query collection for audit events, feed metadata, quarantine records, movements, trace IDs, and object references.
3. Generate a ZIP with manifest.json and JSON exports, store it in object storage, and persist package metadata.
4. Emit audit events for package generation and package retrieval attempts.
5. Add tests or manual validation for date-range, organization, site, product, and trace-ID package generation plus authorization.
<!-- SECTION:PLAN:END -->
