# Data Model

Core entities:

- Organization: tenant boundary for a distributor, facility, or program.
- Site: physical or logical location where inventory is held.
- Medication Product: public reference product, seeded from FDA/openFDA NDC data where possible.
- Lot: product lot with lot number and expiration date.
- Stock Position: current inventory quantity for a site/product/lot.
- Inventory Movement: receipt, dispense, transfer, or correction event.
- Partner Feed: submitted payload from an external system or simulator.
- Quarantine Record: invalid, duplicate, or conflicting payload that requires review.
- Risk Signal: expiration, shortage, or overstock signal generated by rules.
- Audit Event: traceable platform action with actor, source, and trace metadata.
- Evidence Package: exportable review bundle built from audit events and payload metadata.

Phase 1 must avoid patient-identifiable data. Dispense events should be synthetic or aggregated.
